Project Registry

Each application — purpose, stack, and key design decisions

Raspberry Pi · CCTV System

Full-Stack · IoT · Real-Time

End-to-end surveillance system running on Raspberry Pi hardware. Java backend serves AJAX endpoints that the browser consumes for live camera management — rename feeds, toggle fullscreen, and handle authenticated logout without a page reload.

JavaJavaScriptAJAXMySQLCSSApache Tomcat
  • AJAX-driven UI eliminates full-page reloads under live video feeds
  • Session-based authentication gates all camera control endpoints
  • SQL schema stores camera metadata; Java ORM handles query lifecycle

CCTV Dashboard

Admin Panel · Role-Based Access

Administrative control panel for the multi-camera CCTV system. Handles role-based access control, credential management, live connection diagnostics, and persistent Remember Me sessions — all in a pure front-end layer that talks to the Java backend.

JavaScriptHTML5CSS3
  • Role separation: operator view vs. admin view rendered from the same endpoints
  • Persistent session via secure cookie with HttpOnly flag
  • Live connection status polling without WebSocket overhead

Music Galaxy

Full-Stack · Encrypted Data

Full-stack music platform with genre browsing, paginated result sets, and dark/light theming. User data is AES-encrypted at rest. The architecture is designed for Spotify API expansion without schema changes — the data model is provider-agnostic.

PHPMySQLJavaScriptAESbcrypt
  • Passwords hashed with bcrypt; sensitive profile fields AES-encrypted in the database
  • Pagination via SQL LIMIT/OFFSET — avoids loading full catalogue into memory
  • Theme preference stored in localStorage with server-side fallback

The Fly Now

Full-Stack · Booking System

Air travel booking platform with live flight search, account management, booking history, and a fully responsive layout from mobile to widescreen. PHP handles all session state and booking transactions; the frontend is pure HTML/CSS/JS.

PHPMySQLJavaScriptHTML5CSS3
  • Booking transactions use SQL transactions to prevent partial writes
  • Search filters applied server-side to minimize data transfer
  • Session lifetime governed by server-side timeout, not client cookie expiry

HTML & PHP Tools

Toolkit · Rapid Prototyping

A curated collection of full-stack patterns: database-driven tables, validated form processors, and responsive UI components. Each tool is a standalone module that can be dropped into any PHP/MySQL project as a starting point — built to be readable, not clever.

PHPMySQLJavaScriptHTML5PDO
  • All database interaction uses PDO prepared statements — injection-proof by construction
  • Form validation runs both client-side (UX) and server-side (security)
  • Components are stateless modules — no shared global state between tools
03

Security Architecture

Defense applied at every layer — not added after the fact

L1

Input & Transport

Every path from browser to server is hardened before logic executes.

  • XSS prevention — output encoding on all user-supplied strings
  • SQLi prevention — PDO prepared statements throughout; no string concatenation in queries
  • HTTPS / TLS — all traffic encrypted in transit; HTTP connections rejected
  • Input validation — whitelist validation server-side; client-side validation is UX only
L2

Authentication & Session

Access control enforced at the session boundary, not inside business logic.

  • Session tokens — server-generated, rotated on privilege change
  • HttpOnly cookies — session ID inaccessible to JavaScript
  • Role-based access — operator / admin separation; middleware checks on every protected route
  • Timeout enforcement — server-side session expiry, not client cookie TTL
L3

Data Encryption

Sensitive data is protected at rest, independent of database access controls.

  • bcrypt — password hashing with adaptive work factor; no reversible storage
  • AES encryption — sensitive profile fields encrypted at the application layer before write
  • SSL/TLS connections — database connections encrypted end-to-end
  • Minimal privilege — DB users scoped to the minimum operations each application needs
L4

Infrastructure & Forensics

System-level hardening and incident detection capabilities.

  • iptables firewall — allowlist-only inbound rules; default deny
  • Log forensics — journalctl, /var/log/syslog, netstat monitoring pipelines
  • Process auditing — ps aux and grep-based anomaly detection scripts
  • Panoptis 2018 — real-world contribution to Greece's national cybersecurity exercise: Linux forensics and live network traffic analysis
